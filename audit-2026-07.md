# Damask audit — July 2026

> Produced by a 34-agent audit (9 subsystem readers, 9 empirical lens critics
> driving the CLI in sandboxes, 13 adversarially-verified candidate moves,
> 1 synthesis), plus ~750KB of real-world usage data (1,481 facts from a
> production project's edge stores). Every finding below was reproduced, not
> theorized.

## Headline

Damask's write side is architecturally right and agents demonstrably want it —
but the read side lies, the fleet substrate tears data, and the first five
minutes return nothing. Make the loop honest and safe and `damask at` becomes
the reflexive first command of every agent session.

**Motto candidate:** *for the greater good of agents.*

## Thesis

The append-only JSONL + disposable index + meta-edge lifecycle is sound
engineering, and the real-world data proves agents willingly produce
high-quality typed, anchored comprehension. The structural gap: damask is
**write-optimized and read-negligent**. The index computes freshness, ranking,
and relocated anchors for every span, yet every read surface except `at`
ignores all of it. Disputes never suppress (the same fixed bug was
re-investigated 8 times); freshness only ratchets downward with no restore
verb; session identity was silently dead from an env-var mismatch. Beneath
that, the substrate broke its own multi-agent promise (torn parallel writes,
panic on one malformed line, anchors keyed on the knowledge log instead of the
code). None of the fixes require new invention — the machinery for the honest
version already exists in the codebase.

## Killer moves (ranked by leverage-per-effort)

1. **Make every read surface tell the truth** (M) — wire already-computed
   rank/freshness/file:line into `where`/`orient`/`briefing`/`peek`. Every
   edge line becomes `e_x [risk] (0.90) ⚠ src/chat.py:69-213 — summary`.
   Follow-on: `briefing --scope <path> --budget N`.
2. **First five minutes** (M) — flagless init detects `CLAUDECODE` env and
   installs the loop; `record` gains `-m/--summary -c/--confidence` flags
   with clap range validation; `damask bootstrap` deterministically seeds
   manifest/TODO/co-change edges so session 1 reads something.
3. **Never dead-end** (S) — directory rollups on path misses, in-file
   fallback, sanitized FTS, ` AND `-aware predicate errors, unique-prefix ID
   resolution. 0-results must always mean *nothing there*.
4. **The graph self-prunes** (M) — `damask triage` (git-verified bulk-close
   suggestions, never auto), `dispute --reason resolved` routes to `close`,
   `damask confirm <id>` re-anchors stale spans from the already-computed
   relocation.
5. **Anchors track the code, not the knowledge log** (M) — per-source-file
   fingerprint keyed re-resolution; flock'd shared rebuild; GROUP BY the N+1
   loops (warm briefing measured 2.4–4.3s → 0.03s).
6. **Distribution that recruits its own installs** (M) — crates.io (name
   verified free) + prebuilt binaries; the guarded SessionStart fallback
   (shipped, see fix 4) already turns every clone into an advertisement.

## Lesser-model playbook

**Asymmetric literacy:** strong models (or deterministic scans) build and
maintain the graph; weak models consume it through channels requiring zero
recall, zero query-crafting, zero staleness reasoning — and their sloppy
writes are clamped into safe facts rather than rejected or silently accepted
as poison. (1) Consumption is passive, bounded, binary: hooks do all reading;
glyphs and session-counts replace judgment. (2) Writing is one guessable
command with typed guardrails — `damask record file 40 60 risk -m "…" -c 0.9`
mirrors `git commit -m` muscle memory; the harvest Stop-hook nudge only works
if the demanded write is achievable by the weakest model in one attempt.
(3) Misses are loud and every error contains its own copy-pasteable fix.

## Positioning

The comprehension layer for agent fleets: a git-native annotation graph where
what one agent verified — typed, confidence-scored, anchored to file:line,
mechanically freshness-checked — is injected into every future session for
thousands of tokens instead of re-derived for hundreds of thousands. Not
"memory" (vector recall of what was said), not docs (prose that rots unread):
**falsifiability at a location**. The demo is one sentence: *init once, work
normally, and every future session — yours, your teammates', your subagents' —
starts already knowing what the last one learned, minus everything that's
since been fixed.*
